Comenzando con el canvas

Esta sección presenta un simple ejemplo de uso del canvas para mostrar algunos items y manipularlos. Crearemos un canvas en blanco y lo dotaremos de un botón que el usuario puede pulsar para insertar items aleatorios en el canvas. Si el usuario realiza un doble click en un item con el botón 1 del ratón, el color del item cambiará. El usuario puede tambien mover los items alrededor pulsando el botón 1 del ratón y arrastrando. Adicionalmente, los items pueden ser borrados mediante la pulsación del botón 3 del ratón sobre ellos. Los items tendrán un borde ancho cuando el ratón pase sobre ellos, y volverán a tener una linea delgada cuando el ratón les haya dejado.

Presentaremos el programa en pequeñas secciones y explicaremos cada una de ellas por separado.

Example 8-1. Creando la ventana principal y el canvas

Aqui creamos una ventana para el ejemplo del canvas y ponemos un widget canvas dentro de ella. También podemos crear botones que permitan al usuario insertar un nuevo item en el canvas y salir del programa. También definiremos los manejadores auxiliares para las señales de pulsación del ratón del botón de Salida y la señal delete_event de la ventana principal. 

#include <gnome.h>

/* Esto define el tamaño del canvas, en pixels */

#define CANVAS_SIZE 300

/* Prototipos para la funciones que definiremos más tarde */

static void add_object_clicked (GtkWidget *button, gpointer data);
static void exit_clicked (GtkWidget *widget, gpointer data);
static gint delete_event (GtkWidget *widget, GdkEvent *event, gpointer data);


int
main (int argc, char **argv)
{
	GtkWidget *window;
	GtkWidget *vbox;
	GtkWidget *frame;
	GtkWidget *canvas;
	GtkWidget *hbox;
	GtkWidget *button;

	gnome_init ("canvas-example", "1.0", argc, argv);

	/* Crea la ventana principal y la vbox principal */

	window = gtk_window_new (GTK_WINDOW_TOPLEVEL);

	vbox = gtk_vbox_new (FALSE, 0);
	gtk_container_add (GTK_CONTAINER (window), vbox);
	gtk_widget_show (vbox);

	/* Crea una trama para el canvas y el canvas en si */

	frame = gtk_frame_new (NULL);
	gtk_frame_set_shadow_type (GTK_FRAME (frame), GTK_SHADOW_IN);
	gtk_box_pack_start (GTK_BOX (vbox), frame, TRUE, TRUE, 0);
	gtk_widget_show (frame);

	canvas = gnome_canvas_new ();
	gtk_widget_set_usize (canvas, CANVAS_SIZE, CANVAS_SIZE);
	gnome_canvas_set_scroll_region (GNOME_CANVAS (canvas), 0.0, 0.0, CANVAS_SIZE, CANVAS_SIZE);

	gtk_container_add (GTK_CONTAINER (frame), canvas);
	gtk_widget_show (canvas);

	/* Crea la hbox para los botones */

	hbox = gtk_hbox_new (TRUE, 0);
	gtk_box_pack_start (GTK_BOX (vbox), hbox, FALSE, FALSE, 0);
	gtk_widget_show (hbox);

	/* Crea el botón usado para añadir objetos -- pasamos el canvas a 
	/* la llamada en los datos de usuario.

	button = gtk_button_new_with_label ("Add an object");
	gtk_signal_connect (GTK_OBJECT (button), "clicked",
			    (GtkSignalFunc) add_object_clicked,
			    canvas);
	gtk_box_pack_start (GTK_BOX (hbox), button, TRUE, TRUE, 0);
	gtk_widget_show (button);

	/* Creamos el botón usado para salir del programa -- pasamos la
	* ventana principal a la llamada en los datos de usuario.

	button = gtk_button_new_with_label ("Exit");
	gtk_signal_connect (GTK_OBJECT (button), "clicked",
			    (GtkSignalFunc) exit_clicked,
			    window);
	gtk_box_pack_start (GTK_BOX (hbox), button, TRUE, TRUE, 0);
	gtk_widget_show (button);

	/* Conectar a al señal delete_event y ejecutar la aplicación */

	gtk_signal_connect (GTK_OBJECT (window), "delete_event",
			    (GtkSignalFunc) delete_event,
			    NULL);

	gtk_widget_show (window);
	gtk_main ();
	return 0;
}

/* Llamada para la señal de pulsación del botón de Salida */
static void
exit_clicked (GtkWidget *widget, gpointer data)
{
	gtk_widget_destroy (GTK_WIDGET (data)); /* the user data points to the main window */
	gtk_main_quit ();
}

/* Llamada para la señal delete_event de la ventan principal de la
* aplicación */

static gint
delete_event (GtkWidget *widget, GdkEvent *event, gpointer data)
{
	gtk_widget_destroy (widget); /* destroy the main window */
	gtk_main_quit ();
	return TRUE;
}

Como podrás ver, se ha creado un canvas nuevo usando la función gnome_canvas_new(). Entonces establecemos el tamaño incial de la ventana del canvas y la superficie de la región de desplazamiento usando las funciones gtk_widget_set_usize() y gnome_canvas_set_scroll_region() respectivamente. Discutiremos el significado de la región de desplazamiento (scrolling region) más tarde; por ahora, solo la ponemos para ir desde el origen al tamaño del canvas en pixels — usaremos un factor de zoom en el canvas de 1:1, es decir, un pixel por cada unidad.

Example 8-2. Creando items aleatorios en el canvas

Aqui definimos la función de llamada add_object_clicked(), la cuál es usada por el botón "Add an object". Cuando el usuario pulsa este botón, un rectangulo o una elipse será creado usando coordenadas aleatorias. 

  /* Prototipo para el manejador de eventos de los items */

static gint item_event (GnomeCanvasItem *item, GdkEvent *event, gpointer data);

 /*  Llamada para la señal de pulsación del botón de añadir objetos. Crea
  *  un rectangulo o una elipse en una posición aleatoria. El canvas se nos
  *  da en el puntero a los datos de usuario.
  */ 
  

static void
add_object_clicked (GtkWidget *button, gpointer data)
{
	GnomeCanvas *canvas;
	GnomeCanvasItem *item;
	guint type;
	int x1, y1, x2, y2;
	int tmp;

	canvas = GNOME_CANVAS (data);
	
	/* Calcula unas coordenadas aleatorias, con la condición 
	 * que (x1 <= x2) y (y <= y2), y asegurar que los 
	 * objetos no son demasiado pequeños. 
	 */
	 

	x1 = rand () % CANVAS_SIZE;
	y1 = rand () % CANVAS_SIZE;
	x2 = rand () % CANVAS_SIZE;
	y2 = rand () % CANVAS_SIZE;

	if (x1 > x2) {
		tmp = x1;
		x1 = x2;
		x2 = tmp;
	}

	if (y1 > y2) {
		tmp = y1;
		y1 = y2;
		y2 = tmp;
	}

	if ((x2 - x1) < 10)
		x2 += 10;

	if ((y2 - y1) < 10)
		y2 += 10;

	
	/* Escoge un tipo para el item aleatoriamente*/

	if (rand () & 1)
		type = gnome_canvas_rect_get_type ();
	else
		type = gnome_canvas_ellipse_get_type ();

	/* Crea el item y lo hace blanco con lineas negras por defecto.
	 * También, conectarlo a sus señales de eventos, asi podemos conocer
	 * en que momento el item recibe dichos eventos. 
	 */
	 

	item = gnome_canvas_item_new (gnome_canvas_root (canvas),
				      type,
				      "x1", (double) x1,
				      "y1", (double) y1,
				      "x2", (double) x2,
				      "y2", (double) y2,
				      "fill_color", "white",
				      "outline_color", "black",
				      "width_units", 1.0,
				      NULL);
	gtk_signal_connect (GTK_OBJECT (item), "event",
			    (GtkSignalFunc) item_event,
			    NULL);
}

En esta función, primero calculamos algunas coordenadas aleatorias para las esquinas del rectangulo o la elipse que vamos a crear. Tenemos que asegurarnos que x1 y y1 son menor o igual que x2 y y2, respectivamente.

Los nuevos items son creados e insertados en el canvas usando la función gnome_canvas_item_new(). El primer parámetro de esta función especifica el grupo de items del canvas que hará de nuevo padre del item. Para este simple ejemplo, introduciremos todos los items dentro del grupo del canvas de más alto nivel, el item raiz, el cuál obtendremos llamando a la función gnome_canvas_root().

El segundo parámetro de gnome_canvas_item_new() especifica el tipo de item que queremos crear. Esto es simplemente el identificador de tipo en Gtk usado por la clase del item que quieres crear. Escogemos aleatoriamente entre gnome_canvas_rect_get_type() y gnome_canvas_ellipse_get_type(), entonces el pasamos este valor a la función gnome_canvas_item_new().

Los siguientes parámetros son opcionales, y son una lista de pares llave/valor que especifican que argumentos establecer para cada item en particular. Internamente estos son manejados usando el sistema de argumentos de objetos de Gtk. La manera más simple de usar esto es, para cada argumento que quieras establecer, pasa una cadena de caracteres con el nombre del argumento, y el valor que quieres establecer para dicho argumento.

Important: Debes pasar valores con tipo correcto! Recuerda que el compilador de C no puede avisarte de tipos incorrectos cuando usas una lista variable de argumentos.

En este ejemplo, debemos usar tipo double (doble-palabra) para las coordenadas de las esquinas del rectangulo o la elipse. Los colores se pasan usando una cadena con un color valido de la especificación X, y las unidades de anchura se pasan como un double.

Por favor, mira la parte de referencia de la documentación de GnomeCanvas para una información detallada sobre los tipos de argumentos soportados por cada item del canvas.

Pasamos NULL como último argumento para indicar que no tenemos que establecer más argumentos para este item.

Finalmente, conectamos con la señal "event" del item, asi podremos ser notificados cuando el item reciba eventos desde el ratón.

Example 8-3. Definiendo el comportamiento de los items del canvas

Aqui definimos el manejador de eventos para los items que hay en nuestro canvas. Los items de canvas reciben eventos al igual que las ventanas X (X windows). En nuestro ejemplo de manejador de eventos, el usuario puede arrastrar items usando el botón 1. Los items pueden ser borrados pulsando el boton 3 sobre ellos. Si el usuario hace doble click sobre un item usando el botón 1, entonces el color del item cambia aleatoriamente. Finalmente, la anchura del borde de un item cambiará dependiendo de si el puntero del ratón está dentro o fuera del item. 

/* Prototipo de la función que cambia el color del item aleatoriamente */

static void change_item_color (GnomeCanvasItem *item);
/* Llamada para la señales de eventos de los items del canvas. Si el
 * usuario arrastra el item con el botón 1, esté se moverá en consecuencia.
 * Si el usuario hace doble click sobre el item, su color cambiará
 * aleatoriamente. Si el usuario pulsa el botón 3 sobre un item, entoces 
 * el item será destruido. 
 * Cuando el puntero del ratón entra en un item, su ancho de linea se
 * establece a 3 unidades.
 * Cuando el puntero del ratón abandona un item, su ancho de linea se 
 * reestablece a 1 unidad.
 */
 

static gint
item_event (GnomeCanvasItem *item, GdkEvent *event, gpointer data)
{
	static double x, y; /* used to keep track of motion coordinates */
	double new_x, new_y;

	switch (event->type) {
	case GDK_BUTTON_PRESS:
		if (event->button.button == 1) {
			/* Recuerda la posición inicial */			
			x = event->button.x;
			y = event->button.y;
			return TRUE;
		} else if (event->button.button == 3) {
			/* Destruye el item */			
			gtk_object_destroy (GTK_OBJECT (item));
			return TRUE;
		}
		break;

	case GDK_2BUTTON_PRESS:
		if (event->button.button == 1) {
			/* Cambia el color del item */			
			change_item_color (item);
			return TRUE;
		}
		break;

	case GDK_MOTION_NOTIFY:
		if (event->button.state & GDK_BUTTON1_MASK) {
			/* Obten la nueva posición y muevete por la
			 * diferencia  */

			new_x = event->motion.x;
			new_y = event->motion.y;

			gnome_canvas_item_move (item, new_x - x, new_y - y);

			x = new_x;
			y = new_y;
			return TRUE;
		}
		break;

	case GDK_ENTER_NOTIFY:
		/* Establece la linea ancha */		
		gnome_canvas_item_set (item,
				       "width_units", 3.0,
				       NULL);
		return TRUE;

	case GDK_LEAVE_NOTIFY:
		/* Establece la linea estrecha */
		gnome_canvas_item_set (item,
				       "width_units", 1.0,
				       NULL);
		return TRUE;

	default:
		break;
	}

	return FALSE;
}

/* Función útil a la hora de cambiar el color de relleno de un item
 * del canvas aleatoriamente
 */
static void
change_item_color (GnomeCanvasItem *item)
{
	static const char *color_specs[] = {
		"red",
		"yellow",
		"green",
		"cyan",
		"blue",
		"magenta"
	};

	int n;

	/* Coge un color aleatoriamente de la lista */

	n = rand () % (sizeof (color_specs) / sizeof (color_specs[0]));

	gnome_canvas_item_set (item,
			       "fill_color", color_specs[n],
			       NULL);
}

La señal "event" para los items del canvas es muy similar a la señal "event" de los widgets de Gtk. Los manejadores de esta señal obtienen un puntero al item relevante, un puntero al evento que el item ha recibido, y el puntero normal a los datos de usuario. Asi como las señales "event" para los widgets, los manejadores devuelven FALSE si ellos no procesaron el evento o si quieren que se haga otro procesamiento sobre él, o TRUE si manejaron el evento y no quieren ningún que se haga ningún otro procesamiento sobre él.

Nuestro ejemplo de manejador de evento es una sentencia switch que selecciona entre los diferentes tipos de eventos. Tenemos los siguientes casos:

  • Caso en el que el botón 1 se ha pulsado una sola vez, recordaremos esta posición inicial del ratón por si el usuario decide arrastra el item por el canvas.

    Caso en el que el botón 3 se ha pulsado una sola vez, entonces destruimos el item. Esto borrara el item del canvas y liberará su memoria.

  • Cuando el botón 1 se pulsa doblemente, llamamos a la función que aleatoriamente cambia el color del item.

  • Para eventos de movimientos, solo tenemos que manejarlos si el botón 1 está pulsado; esto significa que el usuario esta arrastrando con el botón pulsado. Calculamos las deltas entre la nuevas y la viejas coordenadas del puntero del ratón, y movemos el item en dicha cantidad. Esta no es la mejor manera de hacer posible el arrastar items para todas las aplicaciones, pero es suficientemente buena para nuestro simple ejemplo.

  • Para eventos notificadores de entrada y salida, simplemente cambiamos el ancho de linea del item. La linea se pondrá a 3.0 unidades de grosor cuando el puntero del ratón entre en el item, y se pondrá a 1.0 unidades de grosor cuando el puntero del ratón salga del item.

Como con todos los manejadores de eventos, devolvemos TRUE cuando hemos manejado el evento, o FALSE si no lo hemos procesado.

Notar el uso de la función gnome_canvas_item_set(). Se usa para cambiar los valores de los argumentos de los items. Se asemeja en uso a la función gnome_canvas_item_new(), recibe el item al cuál queremos cambiar los atributos y una lista terminada en NULL de pares llave/valor para los nuevos atributos.
