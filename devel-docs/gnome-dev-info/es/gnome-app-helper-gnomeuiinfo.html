La estructura GnomeUIInfo

GnomeAppHelper utiliza arrays de estructuras GnomeUIInfo para definir los items que componen la barra de menú o de herramientas. Cada item tiene la opción de usar un icono y etiqueta de texto. Cada item tiene que unirse a la llamada de una función para que sea invocada cuando el item sea activado. También los items pueden tener teclas rápidas definidas por ellos. Los items también pueden tener botón de comprobacióno estar agrupados dentro de un grupo de opciones. También los items del menú pueden especificar un subárbol que ellos generarán.

La estructura GnomeUIInfo se define como sigue. 

	typedef struct {
	GnomeUIInfoType type ;
	gchar *label;
	gchar *hint;
	gpointer moreinfo;
	gpointer user_data;
	gpointer unused_data;
	GnomeUIPixmapType pixmap_type;
	gpointer pixmap_info;
	guint accelerator_key;
	GdkModifierType ac_mods;
	GtkWidget *widget;
	} GnomeUIInfo;

Estos son los campos de la estructura GnomeUIInfo.

type

Este campo se utiliza para especificar el tipo de item que debería ser creado. El resto de campos en la estructura definen los atributos de los items. El campo type puede tener los siguiente valores :

GNOME_APP_UI_ITEM

Indica que un item normal debe ser creado. Si el item está dentro de un grupo de botones de opción (ver después), entonces este elemento de opción deberá ser creado, por defecto en su estado inactivo.

GNOME_APP_UI_TOGGLEITEM

Este indica que un item conmutado debe ser creado. Por defecto comienza en el estado inactivo.

GNOME_APP_UI_RADIOITEMS

Indica que un grupo de elementos de opción debe ser creado. El campo moreinfo debe apuntar a un array de elementos que serán creados como elementos de opción en el grupo. Por defecto el primer elemento será el elemento activo dentro del grupo de elementos de opción.

GNOME_APP_UI_SUBTREE

Indica que el elemento creado debe tener un submenú unido a él. El campo moreinfo debe apuntar a un array de elementos que serán puestos en el submenú. Este tipo de elementos no es soportado en las barras de herramientas.

GNOME_APP_UI_SEPARATOR

Indica que un separador debe ser creado. Para los menús este es una línea mientras que para las barras de herramientas es un espacio en blanco.

GNOME_APP_UI_HELP

Indica que entradas de ayuda deben ser creadas. El campo moreinfo debe apuntar a una cadena con la cadena ID de la aplicación. Ver debajo para información adicional sobre como crear entradas de ayuda.

GNOME_APP_UI_JUSTIFY_RIGHT

Indica que todos los items siguientes deben ser alineados en la parte derecha del menú. Solo trabaja con los menús y solo tiene sentido para barras de menú.

GNOME_APP_UI_BUILDER_DATA

Se utiliza para las envolturas de lenguajes y constructores de GUI — no se necesita en los programas normales. Especifica que toda subsecuencia de items debería utilizar una estructura GnomeUIBuilderData diferente de una que estaba en uso cuando el item fue insertado. El campo moreinfo debería apuntar a la estructura GnomeUIBuilderData deseada.

GNOME_APP_UI_ENDOFINFO

Se utiliza para indicar el fín de un array de elementos. Es muy importante incluir un elemento de este tipo como última entrada in arrays de GnomeUIInfo.

GNOME_APP_UI_ITEM_CONFIGURABLE

Utilizado para indicar que los valores para este elemento del menú deben ser cargados del fichero de configuración. Una descripción del formato del fichero de configuración está en the section called Formato de la información para la configuración de las envolturas de teclado(keybindings) para los menús estandar. Esto probablemente no debe ser utilizado manualmente. Cada posible elemento configurable tiene que coincidir con la macro conveniente (ver the section called Distintas macros para rellenar la estructura GnomeUIInfo).

De todas formas, para mayor completitud, se debe dar una descripción del uso en la programación de este tipo. Cuando usamos este item, la mayoría de los datos son ignorados. Las excepciones son tres. Primera, los campos moreinfo y data son tratados exactamente como en un GNOME_APP_UI_ITEM array de elementos GnomeUIInfo. Segunda, el campo accelerator_key es un entero transformado desde un elemento enumerado de GNOMEUIInfoConfigurableTypes describiendo que elemento del menú configurable representa. Finalmente, si el acelerator_key es 0, es decir GNOME_APP_CONFIGURABLE_ITEM_NEW, entonces la etiqueta y el hint (ayuda al pasar el ratón) són ambas utilizadas literalmente.

Los otros campos en la estructura GnomeUIInfo son utilizados para especificar los atributos de los elementos que están siendo definidos.

label

Este campo es utilizado para especificar una etiqueta de texto para el elemento que está siendo creado. La versión no-traducida de la etiqueta debe ser especificada; GnomeAppHelper llama el mismo a las rutinas para la traducción.

Para elementos del menú es deseable tener una letra subrayada en la etiqueta, para que el usuario conozca que el puede presionar la letra en el teclado y activar el correspondiente elemento del menú. Para especificar una letra subrayada, pon (_) antes de la letra deseada de la etiqueta. Ejemplos son, "_Fichero", "_Edición" y "_Copiar".

hint

Este campo especifica una cadena opcional que debe ser utilizada como tooltip en los elementos de la barra de herramientas o en los elementos del menú, poniendolo en la barra de estado. Una versión no traducida debe ser especificada. Si no se desea una cadena de este tipo se debe especificar NULL.

moreinfo

El campo se utiliza para especificar diferentes cosas de acuerdo al tipo de elemento que está siendo definido.

  • Para elementos normales, de opción y de conmutación (GNOME_APP_UI_ITEM, GNOME_APP_UI_ITEM, y GNOME_APP_UI_TOGGLEITEM, respectivamente), este campo debe apuntar a la función que debe ser llamada cuando el elemento es activado. El prototipo general de la función es como sigue :

    void callback(GtkWidget *item, gpointer user_data);

    El argumento item es el item del menú o botón en la barra de herramientas que ha sido seleccionado, y el argumento >user_data es la llamada normal de los datos de usuario.

  • Para submenús ( subtrees, tipo GNOME_APP_UI_SUBTREE), este campo debería apuntar a otro array de estructuras GnomeUIInfo. Este array define el submenú

  • Para elementos de tipo GNOME_APP_UI_RADIOITEMS, este campo debe apuntar a un array de estructuras GnomeUIInfo que define los items que son puestos dentro del grupo de botones de opción. Dentro del array, sólo los elementos de tipo GNOME_APP_UI_ITEM y GNOME_APP_UI_BUILDER_DATA son soportados.

  • Para elementos de tipo GNOME_APP_UI_HELP, este elemento debe apuntar a una cadena que es un ID nombre para la aplicación. GnomeAppHelper buscará el fichero topic.dat en el directorio de ayuda definido por la cadena ID de la aplicación, lo leerá e incluirá la ayuda apropiada en el menú. Este tipo no es valido para las barras de herramientas.

  • Para elementos del tipo GNOME_APP_UI_BUILDER_DATA, este campo debe apuntar a la estructura GnomeUIBuilderData. La información en esta estructura será utilizada por todos los elementos siguientes en el array. Esto es solamente necesario para envolturas de lenguajes y construcción de GUI.

  • Para todos los otros tipos de elementos, este campo es ignorado. Debes ponerlo a NULL por compatibilidad con futuras extensiones de GnomeAppHelper.

user_data

Este debería apuntar a los datos que tú pasas a la llamada que será activada cuando el menú o los elementos de de la barra de herramientas son seleccionados. Frecuentemente dejarás este como NULL, y pasarás un puntero de datos específico a todas las llamadas usando las funciones definidas posteriormente.

unused_data

Este campo está reservado para futuras extensiones, y debe ser puesto a NULL.

pixmap_type

Este debe ser uno de los siguientes valores.

GNOME_APP_PIXMAP_NONE

Especifica que ningún icono debe ser utilizado para este item.

GNOME_APP_PIXMAP_STOCK

Especifica que un icono almacenado debe ser utilizado. El nombre o identificador del icono debe ser especificado en el campo pixmap_info

GNOME_APP_PIXMAP_DATA

Especifica que un icono creado desde un dato XPM debe ser utilizado. Un puntero al dato XPM debe ser especificado en el campo pixmap_info.

GNOME_APP_PIXMAP_FILENAME

Especifica que un fichero con una imagen debe ser cargado y utilizado como icono. El nombre del fichero debe ser especificado en el campo pixmap_info.

pixmap_info

Este campo especifica que la información utilizada para crear un mapa de pixeles para el elemento icono.

  • Para iconos del tipo GNOME_APP_PIXMAP_NONE, debe ser puesto a NULL.

  • Para iconos del tipo GNOME_APP_PIXMAP_STOCK, este campo debe apuntar a una cadena con el identificador del icono almacenado. Para elementos del menú se sugiere que se utilizen los simbolos GNOME_STOCK_MENU_* . Para las barras de herramientas, usa los simbolos GNOME_STOCK_PIXMAP_*

  • Para iconos del tipo GNOME_APP_PIXMAP_DATA, este campo debe apuntar al dato en línea XMP desde el que se creará el icono.

  • Para iconos del tipo GNOME_APP_PIXMAP_FILENAME, este campo debe apuntar a una cadena con el nombre del fichero imagen a cargar. Esta imagen será utilizada como un icono.

accelerator_key

Este campo se utiliza para especificar una opcional tecla aceleradora para el item. Las aplicaciones deberían usar las teclas aceleradoras estandar definidas en las Guías de Interface de Usuario GNOME.

Debes especificar el código de la tecla que tú desees utilizar como aceleradora. Los códigos de las teclas són listados en el fichero include gdkkeysyms.h

ac_mods

Este campo es utilizado para especificar las teclas de modificación que deben ser usadas en conjunción con la tecla especificada en el campo accelerator_key para el elemento acelerador. Las aplicaciones deben usar las teclas aceleradoras estandar definidas en las Guías de Interface de Usuario de GNOME.

El valor en este campo es una máscara de bit compuesta por valores enumerados de GdkModifierType.

widget

Este campo es llenado por la funciones de GnomeAppHelper. Una vez que has llamado a una de las funciones que toman el array de estructuras GnomeUIInfo y lo activas en un arbol widget, los campos widget de lasestructuras en el array tendrán punteros a los widgets creados. Tu puedes usar estos punteros para hacer varias operaciones como ocultar y mostrar los elementos, y hacerlos inactivos a los distintos eventos.

Para las estructuras GnomeUIInfo, GnomeAppHelper proporciona varias macros para poder llenar más facilmente estas estructuras. Hay muchas bastante útiles para elementos que necesitan pocos parámetros y para barras de herramientas, que no tienen muchas opciones como los menús.
