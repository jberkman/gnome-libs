Usando GnomeAppBar directamente

Creando una barra de aplicaciones.

GtkWidget *gnome_appbar_new ( gboolean has_progress , gboolean has_status , GnomePreferencesType interactivity );

GnomeAppBar es una barra de progreso a la izquierda y una barra de estado/minibúfer a la derecha. Puede ser opcionalmente solamente una barra de progreso (sin habla, pero puedes hacerlo), o solo una barra de estado/minibúfer (si no tienes ningún progreso que mostrar).

La barra de estado/minibúfer puede ser una entrada interactiva a lo Emacs o solo mostrar el estado. (De todos modos, por ahora el modo interactivo esta inacabado.) Esto se especifica con el parámetro interactivity, el cuál puede tener tres valores: GNOME_PREFERENCES_NEVER, GNOME_PREFERENCES_USER, o GNOME_PREFERENCES_ALWAYS. Si especificas _NEVER o _ALWAYS, entonces la interactividad se prohibe o se fuerza. Por otra parte, un usuario provisto por GNOME determinará la interactividad.

Usando una GnomeAppBar para mostrar el estado

La barra de aplicación muesta un mensaje cada vez. Hay tres formas de mensaje: mensajes de estado transitorios, mensajes de la pila y mensajes por defecto.

  • Los mensajes transitorios son mostrados, anulando cualquier otro mensaje, pero la barra de aplicación no guarda nada sobre ellos; asi que desaparecen siempre, tan pronto como se refresca la barra de aplicación. La función gnome_appbar_set_status() crea un mensaje transitorio.

  • Si ningún mensaje transitorio lo ha sobreescrito, el mensaje que hay en la cima de la pila se muestra. Las funciones gnome_appbar_push(), gnome_appbar_pop(), gnome_appbar_clear_stack() manipulan la pila.

  • Si la pila está vacia, la barra de aplicación muestra el mensaje por defecto (el cuál puede ser una cadena de caracteres vacia). El mensaje por defecto se establece con gnome_appbar_set_default().

Es posible e incluso estimulante usar solo uno de los 3 tipos de mensaje. Puedes hacer la barra de aplicación tan simple o tan compleja como quieras; con solo gnome_appbar_set_status(), es basicamente solo una GtkLabel.

void gnome_appbar_set_status ( GnomeAppBar *appbar , const gchar *status );

Muchas veces quieres mostrar un mensaje que no es muy importante, y no te quieres preocupar de borrarlo más tarde. gnome_appbar_set_status() hace esto por tí. Pone el mensaje en la barra de aplicación hasta que en un momento posterior la barra cambie.

Si quieres, puedes usar la barra de aplicación con solamente esta función. Para aplicación simples puede ser un buen enfoque.

void gnome_appbar_set_default ( GnomeAppBar *appbar , const gchar *default_status );

Cuando no hay un estado especial que mostrar, la barra de aplicación está normalmente vacia. De todas formas, si lo prefieres puedes poner un mensaje por defecto; este es un mensaje que nunca puede ser eliminado de la pila de mensajes. Para volver a una barra de aplicación vacia, ponemos el mensaje por defecto a una cadena de caracteres vacia.

void gnome_appbar_push ( GnomeAppBar *appbar , const gchar *status );

void gnome_appbar_pop ( GnomeAppBar *appbar );

void gnome_appbar_clear_stack ( GnomeAppBar *appbar );

GnomeAppBar mantiene una pila de mensajes; siempre muesta el mensaje que hay en la cima de la pila. (Hay una excepción: gnome_appbar_set_status() machaca el mensaje de la cima de la pila temporalmente.)

void gnome_appbar_refresh ( GnomeAppBar *appbar );

gnome_appbar_refresh() actualiza la barra de herramientas para reflejar el estado salvado actual. Basicamente esto quiere decir que cualquier mensaje transitorio (creado con gnome_appbar_set_status()) desparecerá para ser reemplazado por el mensaje que haya en la cima de la pila o por el mensaje por defecto.

Usar una Barra de Aplicación para preguntar al usuario

No funciona muy bien, asi que no se documenta. Basicamente puedes presentar un prompt, y capturar una señal si el usuario da una respuesta o cancela. Es util para evitar un díalogo si solo quieres obtener una cadena de caracteres o algo similar. Pero alguien tiene que escribir el widget apropiado, quizás basado en GtkEntry.
