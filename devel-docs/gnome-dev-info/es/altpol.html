Politica sobre retocar el código de otra gente

Si sólo vas a hacer pequeños arreglos, hazlos. Pero si planeas cambiar alguna API o reescribir algún código, por favor, comunicaselo al autor. Los módulos en CVS tienen diferentes niveles de madurez: algunos estan siendo desarrollados activamente, otros son muy estables y sus cambios normalmente deben pasar primero por el mantenedor del módulo. Para ayudar a clarificar esta situación, normalmente, hay un fichero llamado README.CVS incluido en cada modulo del CVS, dónde la política es detallada sobre que retoques puedes hacer a dicho modulo.

Si tienes alguna duda, consulta la lista gnome-hackers@nuclecu.unam.mx
