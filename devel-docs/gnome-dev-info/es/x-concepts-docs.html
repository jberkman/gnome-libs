Chapter 10. Conceptos basicos sobre X

El Sistema X Windows es un gran y compleja maraña. Las librerías Gnome intentan hacer más facil las cosas poniendo una agradable envoltura a dicho sistema. Aún hay unas pocas cosas que deberías tener en mente para poder escribir mejores aplicaciones.

Este capítulo está pensado para programadores con poca experiencia programando las X. Si no conoces que son contextos gráficos, dibujables y visuales, deberías definitivamente leer este capítulo. Si conoces la manera de hacer las cosas con las X, puedes saltarte este capítulo.

Toma nota de que este capítulo está pensado como un pequeño curso de choque sobre los conceptos X que se necesitan tener en mente mientras escribes Aplicaciones de Gnome. Este capítulo no sustituye al manual de las Xlib. Deberías obtener un manual de programación y referencia de las Xlib.

Dibujables

Dibujables son cosas sobre las que tu puedes pintar. Las X tienen dos tipos de dibujables,ventanas (windows) y mapas de pixeles (pixmaps). Los mapas de pixeles son entidades fuera de la pantalla que no puedes ver. Puedes ver cuando esta es mapeada (mostrada) sobre la pantalla.

Las ventanas pueden ser colocadas en una estructura de árbol. Todas las ventanas excepto la ventana raiz tendrán un único padre, pero en cambio todas las ventanas pueden tener varías ventanas hijas. Los mapas de pixeles no se pueden colocar unos dentro de otros, y no pueden tener padres.

Las ventanas tienen las propiedades x/y/ancho/alto que definen su posición dentro de su padre. Los mapas de pixeles solo tienen las propiedades ancho y alto.

Las ventanas y mapas de pixeles pueden tener otras propiedades. Sin embargo, es muy importante la propiedad de visual class de los dibujables, la cual comentaremos a continuación.
