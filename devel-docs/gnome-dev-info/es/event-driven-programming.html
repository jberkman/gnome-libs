Programación de los eventos-conducidos ____

Los programadores que vienen desde el mundo de la consola pueden encontrar la programación GUI un poco estraña al principio. En la programación tradicional, el programa guía al usuario a través de varias pantallas con entradas y opciones, y el programa tiene siempre el control de lo que se hará a continuación.

De la forma GUI, la mayoría de las veces no pondrás el control de flujo de tus programas y dejarás al sistemas que lo conduzca por tí. Ahora es el usuario quien lo controla — cada vez que el programa se ejecuta, el usuario puede elegir una forma diferente de trabajar a través del interface de usuario. Las opciones del menú pueden ser invocadas en cualquier orden y las ventanas también podrán ser abiertas y cerradas.

La programación GUI generalmente trabaja sobre un modelo de eventos-conducidos. Esto significa que las aplicaciones obtienen eventos desde el sistema o del usuario y deben reaccionar a ellas. Esta es una forma diferente del modelo tradicional dónde el programa le dice al usuario lo que debe hacer en el siguiente paso.

Los programas con eventos-conducidos generalmente permanecen alrededor de un bucle de espera buscando eventos por llegar. Cuando un evento es recibido, el programa busca lo que debe hacer y actua en consecuencia. Un evento podría ser "que el usuario presionará un botón del ratón", o "se mueva una ventana", o "salir aplicación (tu debes terminarte)".
