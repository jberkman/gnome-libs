Funciones útiles si un GnomeApp está disponible

Cuando estas funciones devuelven un GtkWidget*, están devolviendo el díalogo creado, si se ha creado alguno. Si no se ha creado ningún díalogo, devuelven NULL. En general esto no tiene ningún uso, pero en circustancias especiales debes tener cuidado. Núnca deberías modificar las caracteristicas del dialgo devuelto, como son: cómo se destruye, dónde se posiciona, etc.; podrías confundir la libreria interna y/o machacar las preferencias del usuario.

GtkWidget *gnome_app_message ( GnomeApp *app , const gchar *message );

gnome_app_message() muesta un mensaje simple de importancia moderada. Vendrá en un díalogo con un botón de Aceptar, o en la barra de aplicación; requerirá que el usuario conozca esto de alguna manera antes de desaparecer.

GtkWidget *gnome_app_flash ( GnomeApp *app , const gchar *flash );

Esta se usa para mensajes triviales que no son muy importantes; un ejemplo podría ser los pequeños mensajes de Netscape, diciendo que URL esta traiendo actualmente. Si el usuario nunca ve estos mensajes, no es un problema; de hecho, gnome_app_flash() no garantiza mostrar el mensaje al usuario (por ejemplo, si los díalogos se cancelan y el GnomeApp no tiene barra de aplicación).

GtkWidget *gnome_app_error ( GnomeApp *app , const gchar *error );

Un error importante, emitido si una operación falla al completo o hay un problema mayor. Esto puede hacer algo molesto como un pitido, y probablemente requerirán el reconocimiento del usuario.

GtkWidget *gnome_app_warning ( GnomeApp *app , const gchar *warning );

Sólo un aviso; no es un error importante, pero no tan trivial como un flash.

GtkWidget *gnome_app_question ( GnomeApp *app , const gchar *question , GnomeReplyCallback callback , gpointer data );

Pregunta la cuestión si o no question, y llama a callback para indicar una respuesta o si el usuario cancela (borrando el díalogo o con control-G desde el eventual widget minibuffer).

GnomeReplyCallback es una definición de tipo: 

	typedef void (* GnomeReplyCallback)(gint reply, gpointer data);
El primer argumento pasado en la llamada es un número representado la contestación del usuario. También puede ser GNOME_YES o GNOME_NO. La llamada también recibe un argumento data, los mismos que pasaste en la función gnome_app_question().

Ten en mente que la llamada no tiene garatizado que sea llamada (N.del.T: O mejor retrollamada). Si el usuario declina responder a la pregunta --- lo dice, destruyendo el díalogo por medio del gestor de ventana --- la llamada es ignorada. Esta es un area para la mejora de las API's; probablemente de la sensación de que hace la llamada contestando con GNOME_USER_CANCELLED o algo así. De todas formas, esta carateristica no está implementada todavía, así que no asumas que tú llamada va a ocurrir.

GtkWidget *gnome_app_question_modal ( GnomeApp *app , const gchar *question , GnomeReplyCallback callback , gpointer data );

Esta función es exactamente la misma que gnome_app_question(), pero crea un díalogo modal o un prompt en la barra de aplicación, así el usuario será incapaz de interactuar con otras partes de la aplicación hasta que se haya ocupado del díalogo (también se contesta a la pregunta, o el usuario cancela cerrando el díalogo/cancela mediante control-G desde la barra de aplicación). Es un defecto esperar a que se arregle el manejar la cancelación de preguntas modales tan pobremente.(Ver también comentario en la introducción acerca de usar posiblemente un GtkObject aqui).

GtkWidget *gnome_app_ok_cancel ( GnomeApp *app , const gchar *message , GnomeReplyCallback callback , gpointer data );

GtkWidget *gnome_app_ok_cancel_modal ( GnomeApp *app , const gchar *message , GnomeReplyCallback callback , gpointer data );

Estas dos funciones son exactamente paralelas a las funciones gnome_app_question(), solo preguntan al usuario Aceptar/Cancelar en vez de Si/No, y la llamada recibe GNOME_OK o GNOME_CANCEL.
