GOAD (Gnome Object Activation Directory) (Directorio de Activación de Objetos Gnome)

Autor(es)

Elliot Lee <sopwith@redhat.com>

Descripción

Un conjunto de rutinas para obtener un listado de los servidores de objetos CORBA disponibles, y conectar a servidores nuevos o ya existentes usando parámetros especificos.

goad_server_list_get - obtiene un listado de objectos de los cuáles sabemos como arrancar.

GoadServer *goad_server_list_get(void);

Descripción

Esta función devuelve un array de estructuras (registros) GoadServer que listan los objetos que el programa puede usar.

Uso

GoadServer *servlist = goad_server_list_get();
	/* usa servlist */
goad_server_list_free(servlist);

goad_server_activate - obtiene el servidor especificado

CORBA_Object goad_server_activate(GoadServer *sinfo, GoadActivationFlags flags);

Descripción

Cuando pasamos la dirección de una estructura GoadServer (una recuperada de goad_server_list_get), esta función intentará también conectar el objeto con un servidor existente, o lanzar uno nuevo.

Uso

GoadServer *servlist, *chosen = NULL;
CORBA_Object dothings;
int i;
servlist = goad_server_list_get();
for(i = 0; servlist[i].repo_id; i++) {
    if(!strcmp(servlist[i].id, "gnumeric")) {
	    chosen = &servlist[i];
	    break;
    }
}
dothings = goad_server_activate(chosen, GOAD_ACTIVATE_SHLIB|GOAD_NO_NS_REGISTER);

Parámetros

  • GoadServer *sinfo

    Una estructura que mantiene información sobre el servidor que a ejecutar.

  • GoadActivationFlags flags

    Las banderas indican cómo quiere la aplicación que se arranque el servidor. Posibles valores (unidos por AND) incluidos:

    • GOAD_ACTIVATE_SHLIB

      Si se necesita crear un nuevo servidor, la aplicación prefiere que sea cargado de una libreria compartida dentro del espacio de direccionamiento de la aplicación.

    • GOAD_ACTIVATE_REMOTE

      Si se necesita crear un nuevo servidor, la aplicación prefiere que se ejecute como un programa separado.

    • GOAD_ACTIVATE_EXISTING_ONLY

      No lanzar un servidor nuevo para la activación especificada - solo comprueba si alguno está en ejecución.

    • GOAD_ACTIVATE_NEW_ONLY

      No comprueba si un servidor existente se está ejecutando para la activación especificada - solamente lanza uno nuevo.

    • GOAD_ACTIVATE_NO_NS_REGISTER

      No registra el servidor que se acaba de lanzar con el servicio de nombres ( sólo valida si el sevidor resulta de ser activado como libreria compartida).

goad_server_activate_with_repo_id - obtiene un servidor que ofrece un interfaz específico.

CORBA_Object goad_server_activate_with_repo_id(GoadServer *server_list, const char *repo_id, GoadActivationFlags flags);

Descripción

Cuando pasemos el ID (identificador) de repositorio de un objeto, esta función intentará también conectar el objeto con un servidor existente que ofrezca dicho interfaz, o lanzar uno nuevo.

Uso

CORBA_Object dothings;
dothings = goad_server_activate_with_repo_id(NULL, "IDL:GNOME/HelpBrowser", 0);

Parámetros

  • GoadServer *server_list

    Un listado de servidores previamente obtenido de goad_server_list_get(), o NULL. Si se pasa NULL, se obtendrá una lista de servidores, se usará, y liberará dentro de la rutina.

  • const char *repo_id

    El ID (identificador) de repositorio de la interfaz que un objeto devuelto debería tener. Acutalmente debe ser la interfaz especificada. En el futuro el Interfaz de Repositorio podrá ser usada para encontrar servidores con interfaces que hereden del interfaz especificado - caveat user.

  • GoadActivationFlags flags

    Ver la documentación sobre los parámetros para goad_server_activate
